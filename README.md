# Классификация состояния кассового аппарата

Небольшой учебный проект по компьютерному зрению.
Модель делает две вещи:
1. Находит кассу на фото (детекция).
2. Определяет состояние кассы: `открыта` или `закрыта` (классификация).

## Что внутри
- `main.py` — обучение, просмотр датасета, запуск интерфейса.
- `prepare_dataset.py` — сборка датасета в формат проекта.
- `dataset/` — итоговый датасет (создаётся скриптом).
- `cash_register_artifacts/` — веса моделей, метаданные, demo-изображения.
- `external_datasets/` — кэш скачанных исходников (если включён сбор датасета).

## Почему выбраны именно эти модели
- `FasterRCNN` для детекции: это надёжный и понятный baseline для задач поиска объекта на изображении. Он стабильно работает на небольших датасетах и хорошо подходит для учебного проекта, где важны предсказуемость и объяснимость.
- `EfficientNet-B0` для классификации: модель достаточно лёгкая, быстро обучается и при этом даёт сильное качество на бинарной классификации. Для задачи `открыта/закрыта` это хороший баланс между точностью и скоростью.
- Связка из двух этапов (сначала ROI, потом класс) выбрана осознанно: классификатор анализирует уже область кассы, а не весь кадр, из-за чего меньше влияет фон и случайные объекты.
- Такой пайплайн проще отлаживать и защищать в курсовой: отдельно видно качество детекции и отдельно качество классификации.

## Быстрый старт

### 1) Установка зависимостей
```powershell
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

### 2) Сборка датасета (без API-ключа)
```powershell
python .\prepare_dataset.py --roboflow-public-url "https://universe.roboflow.com/shazab-amxv6/cash-register-mezqo" --mode replace --target-root dataset --replace-target
```

### 3) Проверка датасета перед обучением
```powershell
python .\main.py --mode preview_dataset --dataset-path dataset --preview-count 120
```

### 4) Обучение
```powershell
python .\main.py --mode train --dataset-path dataset
```

Что важно:
- обучение идёт на GPU, если CUDA доступна;
- лучшие веса сохраняются в `cash_register_artifacts/models`;
- при повторном запуске скрипт использует сохранённые веса и не переобучает всё без необходимости.

### 5) Проверка через интерфейс
```powershell
python .\main.py --mode gui --dataset-path dataset
```

Интерфейс работает с отложенными demo-изображениями, которые не использовались в train.

## Какие файлы с весами важны
- `cash_register_artifacts/models/detector_best.pth` — лучший детектор.
- `cash_register_artifacts/models/classifier_fold_*_best.pth` — лучшие классификаторы по фолдам.
- `cash_register_artifacts/models/model_meta.json` — параметры модели и рабочие пороги.
- `cash_register_artifacts/models/training_summary.json` — краткая статистика обучения.

## Ограничения проекта
- Датасет не очень большой, поэтому на новых условиях съёмки качество может просесть.
- Для более стабильной модели стоит постепенно добавлять новые реальные фото: другой свет, разные углы, разные камеры.
